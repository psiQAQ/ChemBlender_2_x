# ChemBlender 2.2.0 Extension Migration

## Layout

ChemBlender 2.2.0 is extension-only. Runtime source and `blender_manifest.toml` live under `ChemBlender/`; the repository root contains tests, documentation, CI, Agent knowledge, and build artifacts.

Do not copy the extension source into a legacy add-on directory. Build a ZIP and install it into Blender's `user_default` extension repository.

## Local RDKit Wheel

The wheel is not tracked. Download it before validation/build:

```powershell
$wheelPath = 'ChemBlender\wheels\rdkit-2026.3.3-cp313-cp313-win_amd64.whl'
New-Item -ItemType Directory -Path 'ChemBlender\wheels' -Force | Out-Null
Invoke-WebRequest -Uri 'https://files.pythonhosted.org/packages/68/d0/5de3d0d7e66f0e7e7795ab94a53b826e257176c15c9ee79f15621ac040ed/rdkit-2026.3.3-cp313-cp313-win_amd64.whl' -OutFile $wheelPath
$hash = (Get-FileHash -Algorithm SHA256 -LiteralPath $wheelPath).Hash.ToLowerInvariant()
if ($hash -ne 'f8bd59b24e128c9c70c975bfb1920cf610ba3096439a24ca2850eb861e767c48') { throw 'RDKit wheel hash mismatch' }
```

## Runtime Gate

1. Run `blender-mcp --help`.
2. Query Blender version, executable, bundled Python path, runtime system, and extension repositories through MCP.
3. Require Blender 5.1.0 or newer and a host-readable package path.

## Validate and Build

From the repository root, run the extension validator and builder with the MCP-discovered Blender executable. The build step validates first and creates `chemblender-2.2.0.zip`.

```powershell
$pythonBin = 'C:\Program Files\Blender Foundation\Blender 5.1\5.1\python\bin\python.exe'
$blenderBin = 'C:\Program Files\Blender Foundation\Blender 5.1\blender.exe'

& $pythonBin ChemBlender/scripts/validate_extension.py --source-path ChemBlender --blender $blenderBin
& $pythonBin ChemBlender/scripts/build_extension.py --python $pythonBin --blender $blenderBin
```

## Install and Verify

Install using `bpy.ops.extensions.package_install_files(..., repo="user_default", enable_on_install=True, overwrite=True)`.

Verify:

- enabled key `bl_ext.user_default.chemblender`;
- RDKit distribution version 2026.3.3;
- ChemBlender scene/object properties;
- bundled `.blend` node groups;
- two disable/enable cycles;
- properties and preferences entry removed after final disable.

For an automated local lifecycle check:

```powershell
$package = (Get-Item 'ChemBlender\chemblender-2.2.0.zip').FullName
& $blenderBin --background --factory-startup --python tests/blender_smoke.py -- $package
```

Overwriting an extension whose wheel DLLs were loaded by the same Blender process can emit Windows file-lock warnings. Release validation should use a fresh process; CI starts from a clean runner.

## Initial Limitation

The first 2.2.0 package targets Blender 5.1+ on Windows x64 because the pinned RDKit wheel is `cp313-win_amd64`. Other platforms require their own verified wheel matrix before support is declared.
