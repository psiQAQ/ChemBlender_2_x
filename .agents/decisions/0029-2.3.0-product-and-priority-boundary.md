# 0029：2.3.0 以原生格式兼容为主、平台基础为辅

## Status

Proposed for 2.3.0; user direction approved.

## Context

现有 Phase 0–4 已建立广泛语义和可选后端，但基础安装下的真实格式、统一 UI 和保存恢复仍不完整。继续横向增加计算后端会扩大维护面而不形成普通用户闭环。

## Decision

2.3.0 资源按 Native Compatibility 70%、Platform Foundation 30% 分配。基础扩展的核心价值是读取、存储、组织、诊断和可视化已有数据；计算与高级分析只作为可选增强。

正式平台保持 Windows x64、Blender 5.1+。成功以真实文件端到端和用户可操作闭环衡量，不以模型或 adapter数量衡量。

## Consequences

- 暂停将新大型后端列为基础发布门。
- 优先补齐常用原生格式和 Project UX。
- 路线图必须区分 Contract、Synthetic Integration、Real-file Integration、Product UX 和 Release-qualified。

## Rejected Alternatives

- 继续按 Phase 0–4 横向扩展高级功能。
- 将 ChemBlender定位为某一量子软件的计算前端。

## Verification Contract

2.3.0 final必须在仅安装官方 ZIP的环境中完成承诺格式的导入、质量检查、默认视图、项目保存恢复和适用导出。
